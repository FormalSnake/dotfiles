# Edit Template

Guide for editing `.template` files in `~/.config/formalconf/templates/`.

## Template System

Templates contain `{{variable}}` placeholders that get rendered against a theme's color palette. Output goes to `generated/` then is copied to `current/theme/`.

## Variable Syntax

Basic: `{{variableName}}` — replaced with hex color (e.g., `#1E1E2E`)

With modifier: `{{variableName.modifier}}` — replaced with a specific color format

## Color Variables

ANSI colors: `color0` through `color15`
Special colors: `background`, `foreground`, `cursor`, `selection_background`, `selection_foreground`, `accent`, `border`

## Color Modifiers (11 total)

| Modifier | Output Example | Description |
|----------|---------------|-------------|
| (none) / `.hex` | `#1E1E2E` | Hex with `#` prefix |
| `.strip` | `1E1E2E` | Hex without `#` |
| `.rgb` | `rgb(30,30,46)` | CSS rgb() format |
| `.rgba` | `rgba(30,30,46,1)` | CSS rgba() format |
| `.decimal` | `30,30,46` | Comma-separated 0-255 |
| `.r` | `30` | Red component 0-255 |
| `.g` | `30` | Green component 0-255 |
| `.b` | `46` | Blue component 0-255 |
| `.red` | `0.117647` | Red component 0.0-1.0 (6 decimal places) |
| `.green` | `0.117647` | Green component 0.0-1.0 |
| `.blue` | `0.180392` | Blue component 0.0-1.0 |

## Metadata Variables

- `{{theme.title}}` — theme display name
- `{{theme.author}}` — theme author
- `{{theme.version}}` — theme version
- `{{theme.description}}` — theme description
- `{{theme.source}}` — theme source URL
- `{{theme.mode}}` or `{{mode}}` — "dark" or "light"

## Neovim Variables

Available when theme has a `neovim` section:
- `{{neovim.repo}}` — plugin repository (e.g., `catppuccin/nvim`)
- `{{neovim.colorscheme}}` — colorscheme name
- `{{neovim.light_colorscheme}}` — light mode colorscheme (may be undefined)

## Conditional Blocks

```
{{#if variableName}}
  Content shown when variable is truthy
{{/if}}

{{#unless variableName}}
  Content shown when variable is falsy
{{/unless}}
```

Supports dot-notation: `{{#if dark.neovim.light_colorscheme}}`

## Template Types

### Single-mode (default)
One output file per active mode. Filename: `appname.ext.template`

### Partial-mode
Explicitly dark or light. Detected by `-dark.` or `-light.` in filename.
Example: `kitty-dark.conf.template`, `kitty-light.conf.template`
Both variants are generated if the theme has both palettes.

### Dual-mode
Access both palettes simultaneously using `dark.*` / `light.*` prefixes:
- `{{dark.background}}` — dark palette background
- `{{light.accent.rgb}}` — light palette accent in rgb format
- `{{dark.color0.strip}}` — dark palette color0 without #

Must be registered with `"mode": "dual"` in templates.json.
Current dual-mode templates: ghostty.conf, neovim.lua, lynk.css

## Template Manifest

Located at `~/.config/formalconf/templates/templates.json`:

```json
{
  "version": 1,
  "templates": {
    "appname.ext.template": {
      "version": "1.0.0",
      "installedAt": "2026-01-21T00:25:57.019Z",
      "customOverride": false,
      "mode": "dual",
      "targets": ["~/.config/appname"]
    }
  }
}
```

Fields:
- `version` — semver for update tracking
- `customOverride` — if true, CLI won't overwrite on template updates
- `mode` — "single", "dual", or omit for auto-detection from filename
- `targets` — additional directories to copy rendered output to (supports `~`)

## Editing Guidelines

1. After editing a template, set `"customOverride": true` in templates.json to prevent CLI updates from overwriting
2. Output filename = template filename minus `.template` suffix
3. Unknown `{{variables}}` are preserved as-is (not stripped)
4. Conventional comment header: `# {{theme.title}} ({{mode}}) - Generated by FormalConf`

## Constraints

- Do NOT run formalconf CLI commands
- After template edits, remind user to re-apply their theme to regenerate output
- Template engine source: `~/Developer/formalconf/src/lib/template-engine/`
